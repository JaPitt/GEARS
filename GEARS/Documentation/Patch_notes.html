Changes for GEARS v1.2 (20th November 2018)

  1. Bug fix - NRMSE no longer uses the standard deviation weighting.
  2. Complete rework of the html reporting, now includes page based setup with vectorised figures (for figures < 1mb in size)and avoids having to display each figure on the screen to include it in the report as well as improved aesthetics.
  3. A quality check has been added to the bounding procedure that adds robustness to the regularised estimation when the reduced bounds are too tight. This is updated in the documentation.
  4. Warning log has been added, this includes all of the warnings from the procedure and catagorises them based on importance. This can be found in the html report and the results structure.

Changes for GEARS v1.1 (3rd October 2018)

  1. Warning added regarding the condition number of the FIM (both in the code and the documentation).
  2. Bug fix - Parameters that only apear as initial conditions are now recoginsed in the data initialisation.
  3. Bug fix - The typo in the AMICI and MEIGO path error messages has been corrected.
  4. Bug fix - Now parameters associated used only for initial conditions are not required to exist in validation experiment data.
